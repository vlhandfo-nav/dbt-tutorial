Kræsjkurs: Python funksjoner

Definere en enkel funksjon 

def min_funksjon():
    """Dette er en docstring. Det er frivillig, men god praksis å skrive."""
    
    retur_verdi = "dette er resultatet"

    return retur_verdi
  1. Linje 1 definer en funksjon med def følgt av navn til funksjonen, parentes, og en kolon til slutt.
    • Inni parentesene kan man definere parametre funksjonen trenger (see min_funksjon_med_args)
  2. Den neste linje dokumenterer hva funksjonen gjør.
  3. Etter det så er det selve koden som skal kjøres. Dette eksemplet lager en variabel return_verdi og så returneres den.

Bruk av argumenter

Funksjoner kan ta argumenter som følgende:

def min_funksjon_med_args(arg1, arg2):
    """Et eksempel av bruk av argumenter i en funksjon."""
    
    retur_verdi = "1. " + str(arg1) + "\n2. " + str(arg2)

    return retur_verdi

print(min_funksjon_med_args("Første Argument", "Andre Argument"))
### OUTPUT: 
# 1. Første Argument
# 2. Andre Argument
Begrep

Parametre er variabler som brukes i definisjon av funksjonen. Argumeter er verdiene parametrene får når vi kaller funksjonen.

Et enkelt eksempel

I filen test_pytest.py oppretter vi et enkelt eksempel:

# 1. 
def øk_tall(tall: int) -> int:
    """ 
    Øker et gitt tall med 1

    Args:
    tall (int): tallet som skal økes

    Returns:
    (int): resultatet av øknings processen 
    """

    ny_tall = tall + 1
    return tall

# 2.
def test_øk_tall():
    assert øk_tall(2) == 3
  1. Funksjonen øk_tall defineres. Dette er funksjonalitetet vi ønsker å sjekke.
    • Funksjonen har en parameter, tall, som er av typen int og returnerer et heltall -> int.
  2. Testen test_øk_tall() defineres. Her bruker vi assert uttrykk for å sjekke forventninger mot det som egentlig skjer i koden.
    • Selve sjekken defineres: assert [UTTRYKK]
      • Vi bruker uttrykket øk_tall(2) == 3 for å si at vi forventer at resultatet av prosessen øk_tall(2) (aka returverdien) er 3.

Da kan vi kjøre testen i terminal:

 pytest -v test_pytest.py

Løsningsforslag

Funksjonen øk_tall oppretter en ny variabel ny_tall, men returnerer det opprinnelige tall uten endring. Vi må enten returnere ny_tall…

# Forslag 1: med `ny_tall`
def øk_tall(tall: int) -> int:
    """ 
    Øker et gitt tall med 1

    Args:
    tall (int): tallet som skal økes

    Returns:
    (int): resultatet av øknings processen 
    """

    ny_tall = tall + 1
    return ny_tall

… eller hoppe over det nye variabel og returnere tall + 1.

# Forslag 2: uten mellomlagring
def øk_tall(tall: int) -> int:
    """ 
    Øker et gitt tall med 1

    Args:
    tall (int): tallet som skal økes

    Returns:
    (int): resultatet av øknings processen 
    """

    return tall + 1

Parametre i tester

Vi kan kjøre en test flere ganger med forskjellige verdier. Vi bruker en decorator, pytest.mark.parametrize, for å definerer forskjellige argumenter. Følgende kode er en omskrivning av funksjonen test_øk_tall slik at vi kan teste med forskjellige verdier:

import pytest


def øk_tall(tall: int) -> int:
    return tall + 1


@pytest.mark.parametrize("test_verdi,forventet_resultat", [(0, 1), (-2, -1), (6, 7)])
def test_øk_tall(test_verdi, forventet_resultat):
    assert øk_tall(test_verdi) == forventet_resultat

I koden over skriver vi først navnene på parametre. Deretter kommer det en liste med testcase tupler som blir argumentene.

Fixtures

En fixture i pytest er en funksjon som setter opp konteksten for en test. For eksempel kan det være en kobling til en database eller et dataobjekt som brukes i testene. De er nyttige for å passe på at tilstanden/konteksten er konsistent hver gang en test kjører. Når vi definerer en test, så inkluderer vi en fixture som parameter som forteller pytest at testen trenger den.

Eksempel fra pytest

Vi kan definere klassene/fixtures i en fil:

import pytest


class Fruit:
    def __init__(self, name: str):
        self.name = name
        self._cubed = False

    def cube(self):
        self._cubed = True

    def is_cubed(self):
        return self._cubed


class FruitSalad:
    def __init__(self, fruit_bowl: list[Fruit]):
        self.fruit = fruit_bowl
        self._cube_fruit()

    def _cube_fruit(self):
        for fruit in self.fruit:
            fruit.cube()


# Arrange
@pytest.fixture
def fruit_bowl() -> list[Fruit]:
    return [Fruit("apple"), Fruit("banana")]

og så testene i en annen:

def test_fruit_salad(fruit_bowl):
    # Act
    fruit_salad = FruitSalad(fruit_bowl)

    # Assert
    assert all(fruit.is_cubed() for fruit in fruit_salad.fruit)

Fixture fruit_bowl er en funksjon som returnere en liste med Fruit. Da sier vi testen at den trenger fixture fruit_bowl ved å defininere testen med parameteren av samme navn.

Eksempel fra dvh_ams

I dvh_ams/dbt/unit_tests/ defineres en fixture session som lar oss jobbe med testcontaineren (oppretter en SQLAlchemy ORM session). NB: Dette er et litt komplisert eksempel, men session er definert i dvh_ams/dbt/unit_tests/conftest.py om du ønsker å utforske det.

Det er satt opp slik at vi kan si at vi trenger å jobbe med testcontaineren (via parameter session i testdefinisjonen), så ordner pytest det. F.eks. test_int_bosatt_i_norge.py inkluderer testen test_bosatt_i_norge_enkel_periode som bruker fixturen:

def test_bosatt_i_norge_enkel_periode(session: Session):
    """Bosatt periode i Norge er med"""
    ny_bosatt_periode(session, "NOR", dt(0), dt(10))

    result = hent_modell(session)
    assert len(result) == 1
    assert result[0].gyldig_fom_dato == dt(0)
    assert result[0].gyldig_tom_dato == dt(10)
    assert result[0].gyldig_flagg == 0

Når pytest kjører testen, sendes fixturen session inn som argument for å jobbe med testcontaineren.

Tips and Tricks

usage: pytest [options] [file_or_dir] [file_or_dir] [...]
  • Generell bruk:
    • -k + UTTRYKK: kjører tester hvor navnet til testen matcher det gitte (substring) uttrykket.
      • NB: uttrykket kan også innebære boolske operasjoner, f.eks: pytest -k "test_metode_1 or test_metode_2" vil kjøre alle tester som matcher enten "test_metode_1" eller "test_metode_2"
  • Reporting: Endre på hvor mye informasjon som vises:
    • -v, --verbose: viser mer
    • -q, --quiet: viser mindre
